## Proyecto Sprint 1

Diseño y documentación de casos de prueba para validar funcionalidades del sistema.

Contexto

Este proyecto consistió en la elaboración de casos de prueba para validar diferentes funcionalidades de una aplicación. El objetivo fue garantizar que los requisitos definidos se cumplieran correctamente antes de la liberación del producto.

Análisis

Se diseñaron escenarios de prueba considerando entradas válidas e inválidas, verificando el comportamiento esperado del sistema y documentando los resultados obtenidos.

Conclusiones

La ejecución de los casos de prueba permitió identificar oportunidades de mejora y validar el correcto funcionamiento de las funcionalidades evaluadas.
